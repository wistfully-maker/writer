# Skill：世界观碎片化释放

## 技能描述
通过"遇到问题→顺势给规则→留下新疑问"的核心循环，让世界观设定在角色行动中自然呈现，而非旁白灌输。

## 适用条件
- 涉及新设定、新规则、新地点、历史背景的场景。
- 开篇世界观建立阶段。
- 检查是否存在"信息倾倒"时。

## 执行指令（给 Architect/Writer 的指导）
1. **核心循环**：遇到问题 → 需要理解规则 → 顺势给出部分规则 → 留下新的疑问。每一次世界观的披露都必须同时完成"满足好奇"和"激发新好奇"两个动作。
2. **开篇禁忌**：前三章严禁出现任何超过 3 句的连续设定说明。世界观必须通过角色的行动和后果来展示，而非通过旁白或角色讲课来灌输。
3. **信息差设计**：读者的信息量可以略多于主角（上帝视角的焦急期待），但绝不能少于主角（那会让读者感到被蒙骗）。主角的认知升级必须与读者的节奏同步。
4. **碎片粒度**：单次释放的规则信息不超过一个段落。如果一个设定需要更长篇幅才能说清，拆成 3 个以上碎片，分散到至少两章中。

执行边界：
- **严禁**让角色突然无理由地"想到"一个从未提及的设定来解释困局。
- **严禁**在开篇 5000 字内使用"众所周知……"或"在这个世界里……"这类直接介绍性语句。
- **不适用于**：最终决战或全书收尾时，可以加速释放剩余设定，不必再严格碎片化。

## 评判检验点（给 Reader 的检查项）
- [ ] 阅读本章时，我是否因为某个设定的揭示而感到"原来如此"的满足？
- [ ] 本章结尾时，我是否因为某个新出现的规则缺口而产生了新疑问？
- [ ] 如果本章有设定说明，是否超过了一个段落的长度？（超过则标记为"信息倾倒"）
- [ ] 主角的认知是否与我的认知基本同步？（如果我比主角早太多猜到，标记为"上帝视角失效"；如果我比主角迷惑太多，标记为"信息虐读者"）

## 反例警告
- 最低劣形式：开篇序章直接说明"这个世界分为九大境界，每境分三层……"然后正文再也没提到过。
- 后期突然出现"其实还有隐藏的第十境"但没有任何前期伏笔，属于设定欺诈。
