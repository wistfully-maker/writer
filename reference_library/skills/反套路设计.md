# Skill：反套路设计

## 技能描述
让读者以为自己可以预测结果，却被一个"完全合理但没想到"的方向击中。前提是伏笔前置和人物动机驱动。

## 适用条件
- 规划预期之外的情节转折时。
- 读者极易猜到的桥段需要升级时。
- 检查反转是否有伏笔支撑时。

## 执行指令（给 Architect/Writer 的指导）
1. **定义**：反套路不是对套路的简单否定，而是让读者以为自己可以预测结果，却被一个"完全合理但没想到"的方向击中。前提必须是**合理性**和**伏笔**。
2. **伏笔前置**：任何反套路的情节，其关键信息必须在前文中至少出现过一次暗示。让读者回头看时产生"原来早有暗示"的感慨，而非"作者临时改了主意"。
3. **三种反套路触发方式**：
   - **预期颠覆**：读者以为 A 会发生，结果发生了 B，但 B 完全符合人物的欲望和恐惧。
   - **套路升级**：一个熟悉的套路被叠加了一个新的变量，使其产生不同的结果。
   - **视角切换**：换个角色的视角来看同一个事件，揭示完全不同的理解。
4. **频率控制**：反套路是调味料，不是主食。每 10 章最多使用 1-2 次重大反套路。频繁使用会耗尽读者信任。

执行边界：
- **严禁**用推翻已有设定来制造反套路。比如一个被设定为"忠厚老实"的角色突然背叛主角，除非有足够的铺垫。
- **严禁**让人物做出不符合其价值立场的选择来制造意外。反套路必须源于人物的深层欲望，而非作者的意志。
- **严禁**在短时间内（5 章内）连续使用反套路，会导致读者觉得"这书没个准信，什么都不敢信了"。

## 评判检验点（给 Reader 的检查项）
- [ ] 这个反转发生后，我回头看前文，是否能找到至少一条相关暗示或伏笔？
- [ ] 反转发生的原因是否来自人物的欲望/恐惧，而非外部强行干预？
- [ ] 这个反转是否符合世界观规则，没有突然"开挂"的感觉？
- [ ] 本章是否有其他人物或读者早已知道的信息，而我一直被蒙在鼓里？（如果是，标记为"信息欺骗"，而非真正的反套路）

## 反例警告
- 最低劣形式：主角和读者一起被蒙蔽，最后才被告知"其实一切都是XX的阴谋"，但前文毫无痕迹。
- 频繁的反套路会变成另一种套路，让读者麻木甚至厌烦。
